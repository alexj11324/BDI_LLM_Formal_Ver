# PRD: ICML Paper Manuscript Prose Generation

## Overview

本 PRD 是 ICML 论文自动化工作流的**第二阶段**——稿件正文撰写。

**前置条件已满足：** 第一阶段（pre-writing dry run）已全部完成，产出了 5 个 packets 和 4 个 lead artifacts，均保存在 `runs/spawnteam_paper/` 下。详见下文 Key Context 部分。

**本阶段目标：** 基于锁定的 packets、claim ledger、integrated outline，生成可直接进入 LaTeX 的各 section skeleton draft。每个 Task 对应一个独立的稿件组件。

**核心约束（不可违反）：**
1. 不可在同一表/声明中混用 frozen PlanBench snapshot 和 current-mainline numbers
2. 不可将 `_legacy/run_planbench_full.py` 描述为 current-mainline runtime
3. 不可将 TravelPlanner oracle validation 数字合并为 deployment/test-time claims
4. 不可假设 figure scripts 产物为权威证据（需验证 path resolution 和 source scope）
5. TravelPlanner `bdi-repair` 结果的 `summary` 字段非 deployment-aligned；必须显式区分 `summary` 与 `validation_as_test_summary`
